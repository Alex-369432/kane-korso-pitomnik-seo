## Как опубликовать изменения на превью
```
bash tools/deploy-preview.sh
```
Нужен GitHub classic PAT (scope `repo`), сохранённый в `/tmp/gh_token.txt` (или переменная окружения `GH_TOKEN`) — токены недолговечны (7–30 дней), если истёк, попросить у пользователя новый: `github.com/settings/tokens` → Generate new token (classic) → отметить `repo`. Подробности — в самом скрипте `tools/deploy-preview.sh`.

Скрипт: клонирует репозиторий во временную папку → копирует туда текущее содержимое этой папки → правит корневые пути под `/kane-korso-pitomnik-seo/` → добавляет кэш-бастинг к `style.css`/`main.js` → коммитит и пушит. Исходные файлы в этой папке не трогает.

## Финальный перенос на боевой домен (когда будет готово)
Не делать без явного запроса пользователя — это заменит текущий живой сайт на Tilda (`cane-corso-pitomnik.ru`). Когда попросят: отключить/архивировать старый сайт на Tilda → перенести DNS домена на новый хостинг.
