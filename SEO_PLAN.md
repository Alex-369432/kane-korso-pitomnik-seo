База: `SEO_RESEARCH.md`. Гео: Москва и Московская область, Центральная Россия.

**Решения по каннибализации:**
- Главная и `/shchenki/` не спорят: главная закрывает широкий бренд/трастовый интент, `/shchenki/` — узкий транзакционный «что есть в наличии прямо сейчас» с конкретными пометами и статусами.
- `/tseny/` не пересекается с `/shchenki/`: цены даёт по классам и методологии, `/shchenki/` — цену конкретного щенка.
- `/o-porode/*` подстраницы не конкурируют друг с другом — каждая закрывает отдельный явный вопрос.
- Блог не дублирует `/o-porode/*`: «вечнозелёные» справочные статьи о породе — в `/o-porode/`, `/blog/` — про процесс (после покупки, разовые темы).
- `/o-pitomnike/` (история, философия, условия) и `/roditeli/` (конкретные производители) разведены по разным страницам, а не смешаны в одну — у каждой свой основной запрос и объём контента, но они дают друг на друга взаимные ссылки.

## 3. Внутренняя перелинковка

- Хедер: Главная, Щенки, Цены, О породе, О питомнике, Отзывы, Блог, Контакты + телефон/CTA всегда видны.
- Хлебные крошки на всех страницах, кроме главной (с `BreadcrumbList`).
- Главная → анонс-блоки со ссылками на `/shchenki/`, `/tseny/`, `/o-pitomnike/`, `/otzyvy/`, `/o-porode/`.
- Каждая карточка щенка на `/shchenki/` → ссылка на `/tseny/` и `/kak-kupit-shchenka/`.
- `/o-porode/*` перелинкованы блоком «читайте также» + CTA «Смотреть щенков в наличии» → `/shchenki/`.
- `/o-porode/komu-podhodit/` и `/harakter-plyusy-minusy/` → `/kak-kupit-shchenka/`.
- `/roditeli/` двусторонне связана с `/o-pitomnike/` и карточками щенков на `/shchenki/`.
- Блог → релевантный раздел `/o-porode/` и `/shchenki/`.
- `/tseny/` и `/kak-kupit-shchenka/` → `/pitomnik-ili-s-ruk/` (предупреждение про низкую цену/риски), `/pitomnik-ili-s-ruk/` → обратно на `/tseny/`, `/kak-kupit-shchenka/`, `/shchenki/`, `/roditeli/`.
- Футер на всех страницах: полный список разделов + юр. страница + соцсети + NAP (название/адрес/телефон — одинаково везде, важно для Яндекса).

## 5. Контент-принципы (Этап 4)

- Каждая страница — свой ракурс, без копирования одинаковых абзацев между страницами.
- Честный тон: `/o-porode/komu-podhodit/` прямо называет, кому порода НЕ подойдёт — крупная служебная собака, нужен опытный хозяин, пространство, ранняя социализация.
- FAQ на коммерческих страницах — короткий (3-5 вопросов), полный список — на `/faq/`.
- Цены — в рублях, диапазоном, с пометкой «уточняйте актуальную цену на момент обращения». Даты помётов — реальные данные заказчика, без выдумки; когда данных нет — «уточняйте у питомника».
- Все трастовые факты (печать на договоре, чип в базе FCI, T1-тест психики, оценки на выставках не ниже «отлично», Про План, Гексаканивак) — реальные данные заказчика, используются дословно по смыслу, не приукрашиваются и не домысливаются сверх сказанного.

## 6. Коммерческие факторы (Этап 5)

- Прозрачная структура цены по классам (`/tseny/`) вместо «цена по запросу» — прямое конкурентное преимущество (ни у одного проверенного конкурента цены нет).
- Явные условия: договор купли-продажи с печатью питомника, чипирование и регистрация чипа в международной базе данных чипированных животных FCI, вакцинация (Гексаканивак).
- Прозрачный процесс покупки по шагам.
- Реальные фото щенков/родителей — на этапе разработки используются подписанные плейсхолдеры (SVG-заглушки с пометкой), готовые к замене на реальные фотографии заказчика.
- FAQ, отзывы, контакты, карта проезда (плейсхолдер), мессенджеры (WhatsApp, Telegram — реальные контакты заказчика).
- Несколько точек CTA без агрессии, не более 1 sticky-CTA на мобильном.
- Политика конфиденциальности обязательна из-за форм сбора данных.

## 7. Технический план (Этапы 6-7)

- Чистый semantic HTML5, без SPA/фреймворка — весь контент в исходном HTML.
- Один H1 на страницу, логичная иерархия H2-H3.
- `<title>`/`<meta description>` уникальны на каждой странице.
- `rel="canonical"` на каждой странице (self-canonical, без параметров) — устраняет недочёт, найденный у corsoliga.ru.
- Open Graph + Twitter Card на всех страницах.
- JSON-LD: `LocalBusiness` на всех страницах (сквозной, с реальными NAP-данными), `BreadcrumbList` на всех, кроме главной, `FAQPage` на `/faq/` и там, где виден FAQ-блок, `ItemList`/`Product`-подобные данные на карточках щенков, `Article` на статьях блога — стек, которого нет ни у одного конкурента.
- robots.txt открыт для индексации, ссылка на sitemap.xml.
- sitemap.xml — статический, все канонические страницы.
- 404.html — понятная страница с навигацией.
- Единый формат URL: везде trailing slash, без `?id=` и регистрозависимых дублей.
- Изображения: `width/height` (без сдвига layout → CLS), `loading="lazy"` кроме LCP-изображения, честные `alt`.
- CSS/JS — минимальные, без внешних шрифтов с блокирующей загрузкой; системные шрифты.
- Аналитика: пустые явные слоты под Яндекс.Метрику и Google Analytics — без выдуманных ID.

## 8. Yandex-специфика (Этап 7)

- Единые NAP-данные в футере каждой страницы + `LocalBusiness` с `addressRegion: Москва`.
- Явная региональность в title/H1 без переспама.
- Простой, быстрый, не перегруженный сайт (поведенческие факторы).
- Mobile First вёрстка.
- Естественные формулировки ключей, без механических повторов.
- Готовность к подключению Яндекс.Вебмастера/Метрики через конфигурационные плейсхолдеры.
